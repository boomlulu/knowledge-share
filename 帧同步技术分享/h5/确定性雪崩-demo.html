怎么看这个演示
两个世界跑的是逐字节相同的代码，在同一个 JS 引擎里执行——唯一区别是世界 B 的初始位置被加了一个 ε。
① 选 ε=0：两球永远重合，两个校验和逐帧完全一致、永不报警——这证明「同代码 + 同输入 = 同结果」。
② 选 ε=0.0001：开局几秒两球肉眼完全重叠，校验和一致；但中央圆盘是混沌系统（Sinai 台球），每次擦过圆盘都把误差指数放大，某一帧突然劈叉 → 校验和对不上 → DESYNC。
③ 现实里那个 ε 不用你手动注入：不同 CPU / 编译器 / Debug-Release 改变浮点运算顺序，自己就会塞进来一个 0.0001。这就是帧同步必须死磕确定性的原因。
注：本动画的分叉是人为注入 ε、在单一引擎里跑出来的，只负责让你秒懂机制；机制为真的证据见《大纲》附录三柱（混沌理论 + IEEE 754、帝国时代 checksum 原文、我们自己的线上去同步日志）。
