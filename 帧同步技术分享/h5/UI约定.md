# H5 演讲播放器 UI 约定

> 目的：固定播放器骨架、视觉语言和协作接口，后续无论由谁继续写页面，都先遵守这里的约定。

## 产品定位

这个 H5 不是 PPT 复刻，也不是文档阅读器。它是一个“演讲播放器”：

- 第一屏直接进入内容，不做营销式封面。
- 每个镜头服务演讲逻辑：制造悬念、偿还悬念、或把问题升级。
- 视觉负责帮助观众理解工程问题，不堆装饰。
- 讲者备注服务排练，不出现在主舞台里抢注意力。

## 文件结构

- `index.html`：播放器页面结构。
- `styles.css`：全局样式和视觉组件。
- `app.js`：镜头数据、渲染逻辑和键盘交互。
- `确定性雪崩-demo.html`：独立演示页，后续可嵌入播放器或改造成一个镜头组件。
- `references/视觉参考-20260625.png`：当前 UI 方向参考图。
- `UI约定.md`：本文件。

当前阶段保持“无构建工具、无框架、无外部依赖”。这样方便现场机器直接打开，也方便其他协作者快速接手。

## 页面布局

播放器分为四块：

1. 顶栏：分享主题、总览按钮、备注按钮、全屏按钮。
2. 左侧章节导航：按幕跳转，帮助排练时快速定位。
3. 中央主舞台：当前镜头的标题、核心句、视觉区、信息债区。
4. 右侧讲者备注：当前镜头讲述目标、提醒和时间提示。

移动端或窄屏下，右侧讲者备注默认收起，章节导航变成横向滚动。

## 主舞台结构

每个镜头固定包含：

- `sceneKicker`：第几幕 + 镜头编号。
- `sceneTitle`：这一镜头的判断句或冲突句。
- `sceneLine`：现场要打出去的一句核心表达。
- `visualFrame`：画面动作区域。
- `payoffText`：正在偿还哪个信息债。
- `hookText`：下一步欠观众什么答案。

不要把长段解释写进主舞台。主舞台只放观众需要立刻看到和记住的信息。

## 场景数据格式

所有镜头先写在 `app.js` 的 `scenes` 数组里：

```js
{
  id: "03",
  chapter: "第一幕",
  title: "接入工具，两分钟后日志开始狂爆",
  line: "不是我主观怀疑，是不同步检测工具开始报警。",
  visual: "logs",
  payoff: "证明“不是肉眼误差那么简单”。",
  hook: "画面还没穿帮，工具凭什么说它不同步？",
  minutes: "2:00 - 4:00",
  notes: "讲者备注。"
}
```

字段约定：

- `id`：两位数字字符串，方便和分镜文档对齐。
- `chapter`：幕名，用于左侧导航。
- `title`：镜头标题，必须能独立表达冲突或判断。
- `line`：一句核心句，不写成段落。
- `visual`：视觉组件类型，由 `renderVisual` 分发。
- `payoff`：当前镜头还了什么债，必须自洽，不能写“回应上一幕第三个问题”这种需要回翻的表达。
- `hook`：下一镜头的问题或悬念。
- `minutes`：排练时间提示，不要求精确到秒。
- `notes`：讲者备注，可写具体讲述提醒。

## 视觉语言

整体气质：未来极简的游戏同步架构演示板。它要像一张浅色产品原型图：中间是浏览器窗口式演讲播放器，左右是说明和导航侧栏，整体轻、干净、有技术产品感。

颜色：

- 背景：浅灰蓝到白色，允许少量柔和蓝色光感和玻璃面板。
- 主文字：深蓝黑。
- 次文字：灰蓝。
- 关键识别色：蓝色用于“导航/选中/主线”，青色用于“可视化/观测”，绿色用于“通过/修正”，红色用于“报警/风险”。

排版：

- 不使用负字距。
- 不用随视口宽度线性缩放正文。
- 主舞台标题可以大，但控制在 1-2 行。
- 按钮和小面板文字保持紧凑，避免演讲时像后台管理系统。

形状：

- 主面板圆角可以到 14px，普通按钮和功能卡片圆角保持 8px 左右。
- 只给真正的功能区使用边框：舞台、备注、导航、总览、工具面板。
- 不使用复古网格、终端扫描线、黑底控制台风格。
- 视觉组件优先像“浏览器演示页、客户端画面、同步监视器、状态面板、权威节点图”，不要像代码编辑器或服务器后台。

## 交互约定

键盘：

- `←`：上一镜头。
- `→` 或空格：下一镜头。
- `N`：讲者备注显示/隐藏。
- `O`：镜头总览显示/隐藏。
- `F`：进入/退出全屏。

鼠标：

- 左侧章节导航跳到对应幕的第一个镜头。
- 底部左右按钮切换镜头。
- 总览卡片跳到具体镜头。

## 协作规则

后续新增镜头时优先做三件事：

1. 先补 `scenes` 数据，确认钩子链条顺。
2. 再新增 `visual` 类型，视觉只服务这一镜头的讲述目标。
3. 最后补样式，复用现有颜色、间距、按钮和面板结构。

不要做的事：

- 不把播放器改成普通 PPT 翻页。
- 不加入和真实经历无关的案例或虚构数据。
- 不在主舞台写大段技术解释。
- 不为了酷炫动画牺牲演讲逻辑。
- 不引入构建链，除非明确决定要进入工程化阶段。
